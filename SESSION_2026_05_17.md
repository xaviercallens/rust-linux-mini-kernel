## 🔍 Problem Solving

### Issue 1: Docker Build Context Path
**Problem:** `../Cargo.toml` not allowed in COPY command  
**Root Cause:** Docker build context restrictions  
**Solution:** Use repository root as build context, reference files relatively

### Issue 2: Container OS Type Error
**Problem:** `InvalidOsType` error on container creation  
**Root Cause:** Missing `--os-type` parameter  
**Solution:** Added `--os-type Linux` to `az container create`

### Issue 3: Directory Creation Failure
**Problem:** `FileNotFoundError` when creating compilation_fixes/  
**Root Cause:** Parent directory doesn't exist  
**Solution:** Changed `mkdir(exist_ok=True)` → `mkdir(parents=True, exist_ok=True)`

### Issue 4: Hardcoded Workspace Path
**Problem:** Script used `/Users/xcallens/...` inside container  
**Root Cause:** Hardcoded default in argparse  
**Solution:** Default to `os.environ.get("WORKSPACE_ROOT", ...)`

### Issue 5: 404 Deployment Not Found
**Problem:** API calls failing with 404 for `gpt-4` deployment  
**Root Cause:** Hardcoded "gpt-4", actual deployment is "gpt-5.3-codex"  
**Solution:** Support `AZURE_OPENAI_DEPLOYMENT_N` environment variables

## 💡 Key Learnings

### Docker & Azure

1. **Build context matters:** Use repository root, reference files relatively
2. **Environment variables:** Always support env var overrides for flexibility
3. **Container requirements:** Don't forget OS type, resource specs
4. **Error handling:** Create directories with `parents=True` in Python
5. **Deployment names:** Don't hardcode - read from environment

### AI Translation

1. **Specifications are valuable:** Document type mappings for future reference
2. **Iterative fixing works:** 3 attempts per module catches most issues
3. **Rate limiting essential:** 60 req/min per endpoint prevents throttling
4. **Context is key:** Include surrounding code in error fix prompts
5. **Checkpointing critical:** Save progress every N modules

### Git & Security

1. **Credentials outside repo:** Use home directory with restrictive permissions
2. **Multiple .gitignore patterns:** Cover credentials, keys, secrets, build artifacts
3. **Environment injection:** Pass secrets to containers, not in images
4. **Commit messages:** Document what, why, and technical details
5. **Small focused commits:** Easier to review and revert if needed
